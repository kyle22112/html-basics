HTML Basics

This guide will help you in learning the basics of HTML and CSS. Most of the stuff in this tutorial is not the only correct way to do it, just the way that I do it.

What is HTML?

HTML stands for HyperText Markup Language. Contrary to popular belief, it is not a programming language. Programming languages are designed to instruct a computer to perform specific tasks, often involving logic, algorithms, and data manipulation (e.g., loops, conditional statements, variables). Markup languages, on the other hand, are used to structure and present content.

What is CSS?

CSS stands for Cascading Style Sheets. CSS's primary purpose is to define the presentation and styling of documents written in a markup language like HTML. It describes how content should look (colors, fonts, layout).

Next, we will go line by line through the boilerplate HTML to explain everything, and hopefully make HTML a little easier to understand.

Line 1, <!DOCTYPE html>

<!DOCTYPE html> is the required document type declaration at the top of an HTML document that tells web browsers to render the page according to HTML5 standards, ensuring consistent display across different browsers. It's not an HTML tag but an instruction that specifies the document's markup language version, allowing the browser to correctly interpret and layout the content.

Line 2, <html lang="en">

The HTML lang attribute specifies the language of an element's content. When placed on the <html> element as <html lang="en">, it declares that the primary language of the entire document is English.

Line 3, <head>

The <head> element is a container for metadata (data about the document). It typically includes the page title, character encoding, links to stylesheets, scripts, and other meta-information that isn’t directly visible on the web page itself.

Line 4, <meta charset="UTF-8">

This meta tag defines the character encoding for the HTML document. UTF-8 supports almost all characters from all human languages, making it a safe and universal choice for modern websites. It helps prevent display issues with special characters and symbols.

Line 5, <meta name="viewport" content="width=device-width, initial-scale=1.0">

This line is essential for responsive web design. It tells the browser to set the width of the page to the width of the device's screen and sets the initial zoom level to 1.0. This makes your page more mobile-friendly and prevents it from appearing zoomed out on phones and tablets.

Line 6, <meta http-equiv="X-UA-Compatible" content="ie=edge">

This tag is used to force Internet Explorer to use the latest rendering engine available (Edge), rather than falling back to an older, less standards-compliant version. It's mostly for compatibility with legacy versions of Internet Explorer.

Line 7, <title>HTML Basics</title>

This tag sets the title of the webpage, which appears in the browser tab and is also used as the default name when someone bookmarks your page. It helps both users and search engines understand what the page is about.

Line 8, </head>

This closes the <head> section. All metadata and links should go before this line.

Line 9, <body>

This opens the body of the HTML document. All visible content (text, images, links, etc.) that you want to appear on the page goes inside this tag.

Line 10, Inside the <body>…

Here is where your actual page content lives. Things like <h1> for headings, <p> for paragraphs, and other HTML elements that present your content go here.

Line 11, <script src="index.js"></script>

This line tells the browser to load and execute a JavaScript file named index.js. JavaScript adds interactivity and dynamic behavior to your page (like dropdowns, popups, or responding to user input). This line should go right before </body> so that the HTML content loads first before the script runs.

Final Line, </body> and </html>

</body> closes the visible content section, and </html> closes the entire document. Everything that makes up your web page should be nested properly within these opening and closing tags.
